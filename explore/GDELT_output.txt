OBSERVATIONS
------------------------------------------------------------------------

1. RATE LIMITING
   GDELT DOC API rate limits are more aggressive than their documentation
   suggests. Even with 6s delays between requests, we hit 429s on the
   second run immediately. The first run exhausted a short-window quota
   (likely per-IP, per-minute). The API is not viable for polling more
   than ~1 query per 10-15 seconds, and may have a harder session cap.
   Verdict: unsuitable as a real-time polling backbone without careful
   throttling and caching.

2. FIELD AVAILABILITY
   ArtList mode returns only: url, url_mobile, title, seendate,
   socialimage, domain, language, sourcecountry.
   No article body text. No sentiment/tone score in this mode.
   No named entity extraction. No CAMEO event codes in DOC API
   (those are in the separate GDELT Events dataset, different endpoint).
   Verdict: metadata only. Content requires scraping the source URLs.
