# KapturFlow

A local-first news intelligence pipeline that tracks Ohio political figures and
ballot/policy issues: pulls headlines from public RSS/search feeds, classifies
and summarizes them with a local LLM (via [Ollama](https://ollama.com), no
API key required), and generates a 7-day / 30-day digest — critical summary,
analysis, and source-cited talking points — per category, per subject.

Live snapshot: **https://kingenderdragon.github.io/political-future-news/**

## What's tracked

| Group | Subjects |
|---|---|
| **Politicians** | Rep. Marcy Kaptur (OH-9), Sherrod Brown, Dr. Amy Acton |
| **Ohio Issues** | Healthcare policy, SJR10 (Ohio voter ID), COVID-19 (Ohio's pandemic experience & Acton legacy) |

Everything tracked is defined in [`subjects.py`](subjects.py) — see
[Adding a new subject](#adding-a-new-subject) below.

## Pipeline

```
rss_collect.py       → pulls new items from each subject's feeds, dedupes, resolves
                        redirect links (Google News / Bing), stores mediaflow_items*.json
mediaflow_classify.py → tags each item with an arc (category), a 1-sentence summary,
                        and a 1-sentence analysis via a local Ollama model
weekly_digest.py     → per arc, per 7d/30d window: a critical summary, an analysis
                        paragraph, and 3-5 source-cited talking points
generate_static_page.py → renders every subject into one self-contained HTML page
                        (docs/index.html) with a Politicians/Ohio Issues toggle
mediaflow_app.py     → live Streamlit dashboard; same data, runs the pipeline on
                        a background timer (every 5 min collect+classify, hourly digest)
```

Every stage takes a subject slug (e.g. `python rss_collect.py brown`) and
defaults to `kaptur` if omitted. Kaptur's data files are unsuffixed
(`mediaflow_items.json`) for historical continuity; every other subject's
files are slug-suffixed (`mediaflow_items_brown.json`).

## Running it

**Requirements:** Python 3.11+, [Ollama](https://ollama.com) installed and
running locally with a model pulled (default `qwen2.5:14b` — override with the
`OLLAMA_MODEL` env var; `llama3.1:8b` also works but produces lower-quality
digests, see `benchmark_models.py`/`benchmark_results.json`).

```
pip install -r requirements.txt
ollama pull qwen2.5:14b

python rss_collect.py <slug>        # collect
python mediaflow_classify.py <slug> # classify
python weekly_digest.py <slug>      # digest

python generate_static_page.py      # renders kapturflow_snapshot.html for ALL subjects
streamlit run mediaflow_app.py      # live dashboard, all subjects, auto-refreshing
```

To publish a fresh static snapshot to GitHub Pages:
```
python generate_static_page.py
cp kapturflow_snapshot.html docs/index.html
git add docs/index.html && git commit -m "Refresh snapshot" && git push
```

## Adding a new subject

Add an entry to `SUBJECTS` in `subjects.py`: a name/context string for the
LLM prompts, search feeds (Google News/Bing query URLs work well — see the
`_gnews()`/`_bing()` helpers), a keyword relevance filter, and an arc
taxonomy (5-ish categories + `UNMAPPED`) suited to what's actually being
tracked — a sitting officeholder's arcs look different from a ballot issue's.
Add the slug to `SUBJECT_GROUPS` under `politicians` or `issues` (or a new
group — `generate_static_page.py` and `mediaflow_app.py` both read
`SUBJECT_GROUPS` generically). Then run the pipeline stages above for that slug.

## Known limitations

- **LLM output can be wrong.** Summaries, analysis, and talking points are
  generated by a local model and can misstate facts. Every item is cited
  with its original article title and outlet/author — verify against that
  before repeating anything, especially specific numbers or dates.
- **Google News links don't always resolve.** Google's redirect-decode
  endpoint rate-limits (CAPTCHA/429) after heavy use, so a stored `link`
  field can end up pointing at a JS-only interstitial page instead of the
  real article. This is exactly why citations (title + outlet/author), not
  clickable links, are the primary attribution mechanism throughout — see
  `backfill_link_resolve.py` to retry resolving links later once any rate
  limit has cleared.
- **Google/Bing News aggregator results don't include author bylines** —
  only direct outlet feeds (Ohio Capital Journal, Cleveland.com, etc.) do.
  Items from aggregator search feeds cite outlet only.
- **`railway.toml`** deploys the Streamlit app to Railway, but that dashboard
  now depends on a local Ollama server — it won't run classification/digest
  generation on Railway as configured. The static page (`docs/index.html`
  via GitHub Pages) has no such dependency once generated.

## Legacy files

`breaking_news*.py`, `eia_terminal.py`, `mediaflow_chat.py`, and `worker.py`
are unused holdovers from this repo's previous life as an oil-futures/Hormuz
crisis monitor (originally forked from `OwenTanzer/oil-futures`). They're not
wired into `mediaflow_app.py` and can be ignored or removed.
